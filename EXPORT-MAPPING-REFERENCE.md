# Valley Study Behavioral Map — Export Mapping Reference

## Overview

Three exports are generated by `src/lib/exportXlsx.ts` (812 lines):
1. **Region Export** (`exportRegion`) — Multi-sheet aggregate view of all 11 workplace anchors
2. **Workplace Export** (`exportWorkplace`) — Single-anchor or non-anchor place view with inflow/outflow detail
3. **Corridor Export** (`exportCorridor`) — Single corridor with mode/direction/partner filtering

All three exports read from JSON files in `public/data/` that are produced by Python build scripts in `scripts/`.

# EXPORT 1: REGION EXPORT

## Entry Point
`exportRegion(input: RegionExportInput)` (lines 248–346)

**Inputs:**
- `flowsInbound` — FlowRow[] (latest year, from flows-inbound.json)
- `flowsOutbound` — FlowRow[] (latest year, from flows-outbound.json)
- `zips` — ZipMeta[] (from zips.json)
- `racFile` — RAC with aggregate + per-ZIP entries
- `wacFile` — WAC with aggregate + per-ZIP entries
- `corridorIndex` — Map of corridor records
- `flowIndex` — Map of corridor flow entries

**Ignores all user filters** — always exports unfiltered inbound + outbound.

## Sheet Manifest

### Sheet 1: Summary (Lines 251–346)

**Rows Dimension:** Summary metrics (not tabular; key-value structure) + Workplace ZIP Code Rankings table

**Columns:** Metric | Value | Detail (or ZIP | Place | Rankings fields)

**Column Computations:**

| Metric | Value Computation | Source |
|--------|-------------------|--------|
| Total Workforce | `sum(inbound FlowRow.workerCount)` | flows-inbound.json aggregated |
| Cross-ZIP Commuters | `sum(inbound.workerCount where originZip ≠ destZip)` | flows-inbound.json, exclude self-flows |
| Cross-ZIP Share | `crossZipCommuters / totalWorkers` | Derived from above |
| Avg Commute Distance (mi) | `meanCommuteMiles(flowsInbound, zips, driveDistance)` (See flowQueries.ts) | drive-distance.json or Haversine fallback |
| Top Inbound Corridor — Workers | Highest worker count on any corridor via `topCorridorOf()` | corridors.json + flows aggregated |
| Top Outbound Corridor — Workers | Highest worker count on any corridor (outbound mode) | corridors.json + flows aggregated |
| Top OD Pair — Workers | Single FlowRow with highest workerCount | flows-inbound.json |
| Outside 11 ZIP Codes (share) | Workers whose residence ZIP is not in ANCHOR_ZIPS / totalWorkers | computeAggregated() from flowQueries.ts |
| Outside Colorado (share) | Workers with originZip == "ALL_OTHER" / totalWorkers | flows-inbound.json |
| Data Vintage (LODES year) | `wacFile.latestYear ?? racFile.latestYear` | wac.json/rac.json header |

**Rankings Table** (sorted descending by Outbound Commuters):
- **ZIP** — anchor ZIP code
- **Place** — place name lookup
- **Outbound Commuters** — sum of outbound FlowRow.workerCount where originZip == this ZIP
- **Outbound % of Residents** — outbound / (outbound + within-ZIP)
- **Inbound Commuters** — sum of inbound FlowRow.workerCount where destZip == this ZIP
- **Inbound % of Workforce** — inbound / (inbound + within-ZIP)
- **Within-ZIP** — sum of FlowRow.workerCount where originZip == destZip == this ZIP
- **Local Share** — within-ZIP / (inbound + within-ZIP)

**Rows Dimension:** 10 rows per dataset (Total Jobs, Age buckets ×3, Earnings ×3, Industry NAICS-3 ×3)

**Columns:** Dimension | Bucket | WAC (Jobs in 11 anchors) | WAC Share | RAC (Residents of 11 anchors) | RAC Share

| Row | Dimension | Bucket | WAC Computation | RAC Computation |

### Build Chain

**Step 1: Fetch raw LEHD data** (`scripts/fetch-lodes.py`)
- Reads: LEHD API (Census Bureau) for CO state
- Raw files: `data/lodes-cache/raw/co_od_main_JT00_*.csv.gz`, `co_rac_S000_JT00_*.csv.gz`, `co_wac_S000_JT00_*.csv.gz`
- Outputs: 2002–2023 annual extracts (all blocks)

**Step 2: Filter to study region** (`scripts/fetch-lodes.py` post-processing)
- Retains OD, RAC, WAC rows touching the 11 anchor ZIPs or their CO neighbors
- Outputs: `data/lodes-cache/filtered/{od,rac,wac}-YYYY.csv`

**Step 3: Ingest & aggregate** (`scripts/build-data.py` + `lodes.py`)
- Reads: Filtered LEHD CSVs for 2002–2023
- Renames columns: LODES variable codes (S000, SA01, etc.) → human keys (totalJobs, ageU29, etc.)
- **RAC/WAC aggregation** (lodes.py `aggregate_rac_or_wac()`):
  - Groups block-level rows by (ZCTA, year)
  - Rolls up NAICS-20 columns (CNS01..CNS20) into NAICS-3 buckets (goods, tradeTransUtil, allOther)
  - Outputs: Per-ZCTA annual totals for 2002–2023
- **OD aggregation** (lodes.py `aggregate_od_to_zip_pairs()`):
  - Groups by (h_zcta, w_zcta, year) — residence ZIP, workplace ZIP, year
  - Replaces out-of-state ZCTAs with "ALL_OTHER"
  - Outputs: ZIP-pair level totals for 2002–2023

**Step 4: Build export JSONs** (`scripts/build-data.py` main, lines 540–676)
- Builds per-ZIP + aggregate panel objects for RAC/WAC/OD (lodes.py `build_rac_or_wac_entries()`, `build_od_summary()`)
- Per-ZIP latest block: totalJobs + age/wage/naics3/race/ethnicity/education/sex buckets
- Per-ZIP trend: 22-point series for totalJobs, age, wage, naics3
- Writes: `public/data/{rac,wac,od-summary}.json`
- Writes flows (latest year): `public/data/{flows-inbound,flows-outbound}.json`

| 4 | Cross-ZIP Share of Universe | `cross-ZIP total / inbound total` | `cross-ZIP total / outbound total` | Percentage of workers commuting across ZIP line |

#### Section B: Top 10 Origins (Inbound)

**Dimension:** Individual residence places + Within-ZIP row + All Other residual row

| Column | Computation |
|--------|-------------|
| Rank | Ordinal 1–10 |
| Place | Place name (aggregated from multi-ZIP origins via `aggregateByPlace()`) |

**Within-ZIP row** (if > 0): originZip == destZip == selectedZip; workers = self-flow count

Same structure as Section B but reversed:

# PASS-THROUGH DATA (flows-passthrough.json)

## Generation Script
`scripts/build-passthrough.py`

## Source Data
- **Raw LEHD OD main** (2023): `data/lodes-cache/raw/co_od_main_JT00_2023.csv.gz`
- **Statewide block→ZCTA xwalk**: `data/lodes-cache/raw/co_xwalk.csv.gz`
- **ZCTA centroids (longitude)**: 2024 Census ZCTA gazetteer + CITY_CENTROIDS overrides

## Definition

Pass-through commutes for anchor A are OD pairs where:
- Residence ZIP ≠ A AND Workplace ZIP ≠ A
- Residence and workplace **flank A longitudinally** (one east, one west of A's centroid longitude)
- Sorted by worker count, capped at 5000 explicit pairs per mode; excess → residual

## Mode-Aware Bucketing

**Inbound mode** (per anchor A):
- Workplace ∈ {other 10 anchors}
- Residence = any non-A ZCTA on opposite side of A

**Outbound mode** (per anchor A):
- Residence ∈ {other 10 anchors}
- Workplace = any non-A ZCTA on opposite side of A

## JSON Structure

```json
{
  "year": 2023,
  "pairsPerAnchorPerMode": 5000,
  "byAnchor": {
    "81601": {
      "inbound": {
        "pairs": [
          { "originZip": "...", "destZip": "...", "workerCount": N },
          ...
        ],
        "residual": M
      },
      "outbound": { ... }
    },
    ...
  }
}
```

# DRIVE DISTANCE DATA (drive-distance.json)

## Generation Script
`scripts/build-drive-distance.py`

## Source Data
- **OSRM public demo server**: `https://router.project-osrm.org/table/v1/driving`
- **Inputs**: flows-inbound.json, flows-outbound.json, zips.json (centroid coordinates)

## Computation

Iterates every ZIP-pair appearing in inbound or outbound flows:
1. Canonicalize pair to sorted order (smaller ZIP | larger ZIP)
2. Query OSRM table API for driving distance + duration
3. Assume symmetry; store one row per pair
4. Cache results in `.osrm-distance-cache.json`; resume on interrupt

## JSON Structure

```json
{
  "80002|81601": { "miles": 153.2362, "seconds": 10652.9 },
  "80003|81601": { "miles": 157.6827, "seconds": 11003.2 },
  ...
}
```

**Key format:** `smaller_zip|larger_zip`

**Usage in exports:** Mean commute distance = worker-weighted average of drive-distance miles (fallback to Haversine × 1.25 if pair not found).

# KEY FORMULAS & LOGIC

## Flow Aggregation

**Inbound to Anchor Z:**
- Filters: `FlowRow.destZip == Z` AND `originZip ∈ ANCHOR_ZIPS || originZip ∈ {named CO ZIPs} || originZip == "ALL_OTHER"`
- Worker count = sum of matching FlowRow.workerCount
- Self-flow (live & work) = subset where `originZip == destZip == Z`
- Cross-ZIP = total − self-flow

**Outbound from Anchor Z:**
- Filters: `FlowRow.originZip == Z` AND `destZip ∈ {same set}`
- Worker count = sum of matching FlowRow.workerCount
- Self-flow & cross-ZIP computed as above

## Segment Aggregation (Demographics)

Each demographic bucket (e.g., "Age U29") is computed as sum of:
- `FlowRow.segments.age.u29` across all matching flows

Where no segment data exists (flows loaded without segment detail), the column carries blank.

## Corridor Aggregation

Per corridor C, for all FlowRow matching:
- `FlowRow.corridorPath.includes(C)` AND
- Active mode/direction/partner/segment filters (except segment filter is ignored per spec)

Worker count = sum of workerCount; origin/dest ZIP aggregation via groupby.

## Cross-ZIP Share Calculations

- **Share of total** = workers / (total inbound or outbound, depending on context)
- **Percentage** formatted as `0.0%` Excel format (0.5 → 50.0%)
- **Denominator**: Total inbound (for inbound flows) or total outbound (for outbound); includes self-flows in denominator

# DATA QUALITY & VALIDATION

All numbers are integer worker counts except:
- **Percentages/shares** are fractions (0.0–1.0) formatted as percentages
- **Drive distance** in miles (floating point, 4 decimals)
- **Corridor length** in meters (floating point, 1 decimal)

Per `build-data.py` reconciliation (lines 552–617):
- Inbound flow total should equal WAC latest total ±0.5% (INBOUND_DRIFT_TOL_PCT)
- Outbound flow total may exceed RAC total (residents working out-of-state excluded from CO-only OD pulls)
- Segment axis sums within ±2 of totalJobs per flow row (LODES noise infusion)
